note: 
模块进行解耦 可以任意选择模型
updated:
1.返回统一使用SequenceClassifierOutput进行封装，方便送入Trainer

2. 模块进行解耦 可以任意选择模型  
  - RecurrentEncoderBlock 复用了三种 RNN cell，通过 _CELL_REGISTRY 内部再做一次映射，而不是三个类
  - 每个 block 暴露 output_size 属性，TorchModel 用它构建 Linear 层，不需要关心内部维度
  - 新增模型（如 CNN、Transformer）只需：① 写一个新的 EncoderBlock 类，② 在 ENCODER_REGISTRY 里加一行，其他代码零改动

  如需要添加新模型
  -   第一件事：在 model.py 里写新的 EncoderBlock 类
  必须满足的两个约定：

  ┌───────────────────────────────────────────────────┬─────────────────────────────────────────────────────────────┐
  │                       约定                        │                            原因                             │
  ├───────────────────────────────────────────────────┼─────────────────────────────────────────────────────────────┤
  │ self.output_size 属性                             │ TorchModel.__init__ 用 self.encoder.output_size 构建 Linear │
  ├───────────────────────────────────────────────────┼─────────────────────────────────────────────────────────────┤
  │ forward(input_ids, attention_mask=None) -> (B, H) │ TorchModel.forward 固定用这个签名调用                       │
  └───────────────────────────────────────────────────┴─────────────────────────────────────────────────────────────┘
  - 第二件事：在 ENCODER_REGISTRY 里注册一行

  ├── hp_search()        # 模块A：单模型超参数搜索

  │     ├── 逐一调用 main(cfg)，异常不中断

  # 只对比模型（默认Config）
  python run_experiment.py --mode model_compare --models roberta lstm gru rnn

  # 完整pipeline：先搜HP，再用最优HP对比
  python run_experiment.py --mode full --model roberta --models lstm gru rnn --max_trials 20

  ---
  第二阶段：单次 trial main(cfg)

  main(cfg)
  │
  ├── 1. 设置随机种子（每次 trial 重置，保证独立可复现）
  │
  ├── 2. load_datasets(config)          ← loader.py
  │   └── AutoTokenizer.from_pretrained(Qwen路径)
  │       tokenizer.pad_token = eos_token  ← Qwen 无 pad_token，此处补上
  │
  ├── 3. TorchModel(config)             ← model.py
  │   └── build_encoder(config)
  │       └── ENCODER_REGISTRY["qwen_lora"]
  │           └── QwenEncoderBlock(config)
  │               └── AutoModel.from_pretrained(Qwen路径, torch_dtype=float16)
  │                   encoder.output_size = 3584  (Qwen2.5-7B hidden size)
  │   └── classify = Dropout + Linear(3584 → 3)   ← float32
  │
  ├── 4. apply_lora(model, config)      ← lora_optimize.py
  │   └── LoraConfig(r=cfg["lora_r"], lora_alpha=cfg["lora_alpha"], ...)
  │   └── get_peft_model(model.encoder.encoder, lora_cfg)
  │       └── 冻结全部 Qwen 权重，只在 q/k/v/o/gate/up/down_proj 插入 LoRA 矩阵 A、B
  │       └── print_trainable_parameters()  → 约 0.1% 参数可训练
  │
  ├── 5. TrainingArguments
  │   └── fp16=True, save_strategy="no"
  │
  ├── 6. CustomTrainer.train()
  │   └── forward 路径：
  │       input_ids → QwenEncoderBlock.forward()
  │           └── self.encoder(input_ids)  ← PEFT 包装后的 Qwen，只有 LoRA 矩阵更新梯度
  │           └── mean pooling → float32  (B, 3584)
  │       → TorchModel.classify(x)        ← Linear(3584→3), float32
  │       → CrossEntropyLoss
  │
  ├── 7. trainer.evaluate() → metrics dict
  │
  └── 8. save_lora_adapter(model, model_path/adapter)   ← lora_optimize.py
      ├── model.encoder.encoder.save_pretrained()
      │   └── 只写 adapter_config.json + adapter_model.safetensors  (~几十MB)
      └── torch.save(model.classify.state_dict(), "classify_head.pt")
